Assunto: Projeto de software para compactação de arquivos.

A descrição completa do projeto se encontra em http://giulianoxt.googlepages.com/rebhu.

Para compilar (assumindo shell no diretório do projeto):
cd Rebhu-qt4
make
(Caso o Makefile default não sirva: algum erro de compilação)
qmake Rebhu-qt4.pro
make

Requisitos:
Qt 4.3.2 (junto com o Qt Designer 4.3.2)
Compilador C++ com implementação da STL.

Navegação no projeto:
O website do projeto conta com ótimas descrições do projeto em geral. Recomendo começar por lá caso queira se inteirar do projeto como um todo. Depois, entre na seção de documentação Doxygen (link Documentação Online) para ter uma visão mais técnica dos projetos (todas as classes e a maioria das funções estão documentadas lá. Tudo com código fonte junto, para acompanhar a documentação).

Como o projeto tomou dimensões acima do esperado, vai uma lista abaixo, header por header, explicando a função dos elementos presentes em cada um:

Classes referentes à GUI em Qt:
rebhuMainWindow.h: Modificação de uma QMainWindow que representa a janela inteira da GUI do Rebhu. Contêm toda a lógica da GUI junto com a interação com um objeto da classe RebFile, presente em RebhuMain.h.
rebhuAbout.h: Modificação de uma QDialog com o tema do Rebhu, informações gerais sobre o software.
rebhuProgressDialog.h: Modificação de uma QDialog com o tema do Rebhu e com informações de progresso referentes à compactação de um arquivo.

Classes em geral do Rebhu:
RebhuMain.h: Arquivo principal da biblioteca Rebhu. Contêm as funções de compressão e descompressão além da classe chave RebFile que simula a iteração com um arquivo .reb aberto. Esta classe utiliza quase todas as outras classes do Rebhu, com exceção das classes da GUI.
RebhuDefaults.h: Contêm duas exceções praticamente não utilizadas e a classe UInt responsável por fazer conversões entre unsigned int e byte array (byte*). Utilizada por algumas classes dos algoritmos de compactação, porém é obsoleta: a classe BitArray já tem métodos que realizam essas operações.
BitArray.h: Contêm a classe BitArray, modelando um bitset da stl com mais funcionalidades.
BitFile.h: Classe que representa um arquivo que pode ser escrito/lido bit a bit ou byte a byte, dependendo da necessidade.
BinarySearchTree.h: Implementação direta de Árvore Binária de Busca.
BlockSort.h: As funções deste arquivo não estão sendo utilizadas no projeto. São algoritmos experimentais de ordenação para o BWT, mas que não atinjiram resultados melhores que as funções de ordenação padrão.
BurrowsWheeler.h: Codificador/Decodificador do algoritmo de transformação BWT.
Huffman.h: Codificador/Decodificador do algoritmo Huffman.
HuffmanTree.h: Subclasse de BinarySearchTree<byte>, modela uma árvore com peso (árvore de Huffman), utilizada para gerar os códigos deste algoritmo.
PriorityQueue.h: Implementação direta de uma fila de prioridade, utilizada no Huffman.
Rle.h: Codificador/Decodificador do algoritmo de compactação Run Length Encoding.
